Wheel of Fortune

Final project of Python Classes and Inheritance from University of Michigan, with some personal modifications for a more user friendly playable game

In this game the player will play against the Computer Player, the user enters the name of the players, the program will choose a random category and a random phrase and the game starts.

The player will spin the wheel and get a random prize, if they prize is bankrupt player loses all their money and the turn

The player will input in a textfield the letter they think it appears in the phrase, vowels have a cost of $250, so, unless the player has a balance equal or superior to that number they won't be able to input vowels. If the user inputs a letter that exists in the word they will gain as many money the wheel indicated multiplied for the number of occurrences of said word. For example:

PHRASE: You Gotta Fight Now
PRIZE IN WHEEL: 500$
LETTER GUESSED: T

The number added to the balance will be: 2*500 = 1000

If the player guesses the word correctly they get to keep the turn, otherwise it will be the other player's (CPU). Given that the player has the advantage of guessing the whole word (which CPU hasn't), CPU doesn't have the possibility of going bankrupt or losing a turn in the wheel, so the game gets balanced a little bit.

File system and signal management

Using UNIX system calls and tools, this console application called els (enhanced - ls) prints report files with the most important information of directories and subdirectories where the application is located

Information shown in each file:

  • Absolute path of the directory
  • Permissions, as they are shown when you execute the following command ls -l
  • user-id, group-id from the directory's owner
  • Last modified date and last access date
  • Number of files contained in the directory (including subdirectories)
  • Number of total bytes occupied by the files contained in that directory (does not include subdirectories)

C Programming Exercises

Various basic algorithm's exercises written in C. Compiled with Visual Studio.

Exercises:

  • Combine two words: The program will ask the user to input text 1 and text 2 and it will return a combination between the words letter by letter, for instance
  • Sum the digits consecutively: Sum digits and if the result has more than 1 digit sum the digits again
  • Break down a number: Break down a number by its digits
  • Common digits: Given two numbers print a third one with the common digits between the first and the second number. Example: Given 721 and 717 the output of the program will be: 71. Do not repeat the numbers in the generated one.
  • Crypting: Given a word encode it at n positions forward in the alphabet. Example: Given the word hello and n = 2 the output of the program should be jfnnq, with the same word and n = 1 the output would be ifmmp
  • One dimension array average and maximum: Fill an one dimension array of 4 elements and calculate the average and the maximum value of it
  • Transposed matrix: Given a matrix (2 dimension array) calculate its transposed
  • Min number: Given a matrix calculate the minimum value
  • Matrix multiplication
  • Grouping: Given an unidimensional array leave the even numbers at the beginning and the odd numbers at the end (same order they appear)

Linked lists exercises:
  • Insert element at the beginning and the end of the list
  • Insert at the end of the list without repeating: Insert an element at the end of the list that doesn't exist in the list yet, if it already exists, throw an error
  • Insert element neatly and without repeating: Given this linked list: [4]->[17]->[32]->[84] by inserting the element 19 in it, the output of the program should be: [4]->[17]->[19]->[32]->[84]
  • Insert element before and after a given element
  • Delete element: Delete an element, delete repeated elements and delete non composite numbers
  • Invert list: Given this linked list [4]->[17]->[32]->[84] the output of the program should be [84]->[32]->[17]->[4]
  • Show list

Earthquake

Simple earthquake simulation, each of the given 1..9 regions contains 2 matrixes, one for the possible economic loss and another for the possible number of deaths after the earthquake, the program will calculate the total cost/# of victims and other relevant data.

Requeriments:

  • The territory given will be a record with 1 to 9 regions. Each geographic region is represented by a matrix (M x N) where each item inside represents a geographic grid. Each region doesn't have more than 900 grids (M = 30 and N = 30 as maximum)
  • Each earthquake has an epicenter, which is the spot where it is iniciated, the epicenter will be represented by a geographic grid [i, j].
  • Each geographic region has 2 matrixes M x N, one represents COST. each grid represents economic losses to consider if an earthquake attacked given zone. There's as well a matrix that represents DEATHS, where the estimation of human losses will be saved. This data is rounded up and expressed as integer numbers.
