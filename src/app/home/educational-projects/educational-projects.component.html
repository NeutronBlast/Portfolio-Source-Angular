Pipes and concurrent programming

Console application that applies crypting and decrypting using processes and pipes

Goals:

  • Use the primitives fork, wait, exec, exit for concurrent process management
  • Use pipes as communication mechanism between processes
  • Use primitives to measure the execution time between processes/threads
  • Use primitives to manage either binary or text files

x86 ASM Project

Console application controled by the keyboard's arrows, with the left and right arrows you advance or go behind, with the up arrow you avoid the obstacles that will be appearing in the screen as the user advances

Project details:

  • To calculate the position of every obstacle it will be applied a pseudo random number generator algorithm
  • When an obstacle is avoided, the program will calculate the angle of turn of the wheel using the following formula [(a,b)*(a-1,b-1)]*10 mod 90

P2Bank

The main idea of the system is that a person associated to this is able to send and receive money as long as the other person is also affiliated. All the data is loaded through a text file (.TXT), there will be used linked lists to manipulate the information and the modified data will be saved to the same text file when the execution of the program is over

Requeriments:

  • Data that will be registered to the system: ID, full name, address, phone number, associated bank accounts
  • Bank accounts: Each bank account contains an ID, a bank ID, type of account, balance
  • Transactions: Each transaction contains an ID, amount, type of transaction (payment, charging), date, ID of the person that will receive the transaction, description (40 characters maximum)

Earthquake

Simple earthquake simulation, each of the given 1..9 regions contains 2 matrixes, one for the possible economic loss and another for the possible number of deaths after the earthquake, the program will calculate the total cost/# of victims and other relevant data.

Requeriments:

  • The territory given will be a record with 1 to 9 regions. Each geographic region is represented by a matrix (M x N) where each item inside represents a geographic grid. Each region doesn't have more than 900 grids (M = 30 and N = 30 as maximum)
  • Each earthquake has an epicenter, which is the spot where it is iniciated, the epicenter will be represented by a geographic grid [i, j].
  • Each geographic region has 2 matrixes M x N, one represents COST. each grid represents economic losses to consider if an earthquake attacked given zone. There's as well a matrix that represents DEATHS, where the estimation of human losses will be saved. This data is rounded up and expressed as integer numbers.
